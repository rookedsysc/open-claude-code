# Style And Conventions
- Follow `.claude/core/CODE_PRINCIPLES.md`.
- Prefer clear, explicit naming over generic names; avoid vague suffixes like Info/Data/Helper/Manager unless framework conventions require them.
- Keep implementations simple (KISS) and avoid speculative abstractions (YAGNI).
- Comments and user-facing explanations should be in Korean when added, and comments should be natural prose rather than numbered steps.
- Use ASCII by default unless a file already contains non-ASCII content; docs translations already contain Korean/Chinese text.
